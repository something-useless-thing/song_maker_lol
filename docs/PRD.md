# 📑 [PRD] 제품 요구사항 정의서 (Lean & Core)

## 1. 문서 개요
*   **프로젝트/기능명:** Music Maker — 브라우저에서 바로 만드는 미니 작곡 툴
*   **작성일:** 2026년 07월 20일
*   **현재 상태:** 📝 초안 (Draft)

---

## 2. 배경 및 비즈니스 목적 (Why)

*   **문제 정의 (Problem):** 악기나 작곡 프로그램(FL Studio 등)이 없어도 멜로디나 비트를 즉흥적으로 만들어보고 싶은 사람이 많지만, 진입장벽(설치, 학습곡선) 때문에 시도조차 안 해보는 경우가 많다.
*   **기대 효과 (Impact):** 설치 없이 브라우저에서 바로 그리드를 클릭하는 것만으로 멜로디/비트를 만들고 재생할 수 있게 해서, 음악 창작의 진입장벽을 크게 낮춘다.

---

## 3. 사용자 스토리 (User Story)

*   **As a (누가):** 악기는 없지만 음악 만드는 데 관심 있는 유저인 내가
*   **I want to (무엇을):** 브라우저에서 그리드를 클릭해서 멜로디를 만들고, 여러 악기 소리로 바꿔가며 들어보고 싶다
*   **So that (왜):** 전문 장비나 프로그램 없이도 즉흥적으로 내 아이디어를 소리로 확인해보고 싶기 때문이다.

---

## 4. 핵심 기능 요구사항 (What - MVP Spec)

| 기능 ID | 요구사항명 | 상세 규칙 (Business Logic) | 우선순위 |
| :--- | :--- | :--- | :--- |
| **REQ-01** | 스텝 시퀀서 그리드 화면 (피아노 롤) | 세로축 음높이 × 가로축 박자로 이루어진 그리드. 칸을 클릭하면 노트가 켜지고, 재생 버튼을 누르면 왼쪽에서 오른쪽으로 커서가 이동하며 소리가 재생됨(Tone.js 사용). 음이름(C7, B6...)이 좌측에 항상 표시됨. | **P0** |
| **REQ-02** | 악기 선택 기능 | 화면 상단 트랜스포트바의 세그먼트 토글로 악기(피아노/드럼 등)를 바꿀 수 있음. 선택한 악기에 따라 그리드에서 재생되는 음색이 바뀜. | **P0** |
| **REQ-03** | 재생 컨트롤 | 재생/정지 버튼, 템포(BPM) 슬라이더 + 직접 입력 가능한 숫자 필드, undo/redo, 저장 버튼 제공. | **P0** |
| **REQ-04** | 플레이리스트(패턴 타임라인) | 만든 패턴(마디)을 트랙별로 배치하고, 여러 패턴을 순서대로 이어 붙여 하나의 곡으로 구성하는 타임라인 화면 (FL Studio Playlist의 미니 버전). 트랙별 색상 구분 필요. | P1 |
| **REQ-05** | 저장/불러오기 | 만든 곡(패턴 조합)을 로그인 없이 로컬에 저장하거나, 공유 가능한 링크로 내보내기. | P1 |

## 5. 제외 스코프 (Out of Scope)

*   실제 오디오 파일(mp3/wav) 녹음 및 다운로드 기능
*   다중 사용자 실시간 협업 작곡 기능
*   VST/외부 플러그인 연동, 고급 믹싱(EQ, 컴프레서 등)
*   채널 랙(악기별 상세 설정) 화면 — 헤더 nav에서 제외, 필요해지면 별도 스코프로 논의
*   결제/구독 기능 (이번 3일 프로젝트는 수익모델 없이 포트폴리오용)

---

## 6. 기술 참고 사항

*   사운드 재생은 Tone.js(Web Audio API 래퍼) 사용 — Tone.Transport 스케줄러로 타이밍 관리, setTimeout 기반 타이밍 금지.
*   악기 음원은 기본 신스 대신 **직접 보유한 샘플(wav) 파일**을 사용. `Tone.Sampler`로 음높이별 wav 파일을 로드해서 그리드의 각 노트에 매핑.
*   REQ-04(플레이리스트)는 시간이 부족하면 과감히 P1으로 미루고, REQ-01~03(피아노 롤 그리드 + 악기 선택 + 재생)만으로도 배포 가능한 MVP가 완성됨.
*   디자인 스펙은 `docs/DESIGN.md` 참고 — 헤더/트랜스포트바 레이아웃, 컬러/타이포 토큰 정의돼 있음.
